üìù What is a Diff?

A Diff (difference) is a data comparison calculation that outputs the differences between two files. It is the core algorithm behind version control systems like Git.

In a typical "Unified Diff" view:

  • Green (+) lines represent additions.
  • Red (-) lines represent deletions.
  • White lines are context (unchanged text) to help you locate where the change happened.

üí° Code Review Best Practices

1. Self-Review First

Always diff your own code before submitting a Pull Request. You'll often spot debug statements (console.log), trailing whitespace, or commented-out code that shouldn't be committed.

2. Focus on "Why", not just "What"

The diff shows what changed. Good commit messages and code comments should explain why it changed.

3. Watch for Whitespace

If a diff shows the entire file as changed, you might have changed the indentation (e.g., tabs to spaces). Use the "Ignore Whitespace" option to see the actual code changes.

No. ToolBit runs 100% client-side. Your code never leaves your browser window, making it safe to paste snippets for comparison.

What is a "Unified Diff"?

Unified Diff is a standard format used by Git and patch files. It combines (unifies) the old and new text into a single chronological stream, using + and - to mark changes. It allows you to share changes as a single text file (a patch).
