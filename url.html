‚ñ∏ Understanding URL Encoding

üîó What is URL Encoding?

URL encoding (percent-encoding) converts special characters into a format that can be transmitted over the internet. Spaces become %20, @ becomes %40, etc.

It's also called "percent encoding" because special characters are represented as % followed by two hexadecimal digits.

üéØ When to Use URL Encoding

  • Query parameters: q=hello world ‚Üí q=hello%20world
  • Special characters: @, #, &, =
  • Form data: Submitting forms with special characters
  • API requests: Passing parameters with spaces or symbols
  • Passing URLs: Using URLs as parameter values

‚ö†Ô∏è Reserved Characters

These characters have special meaning in URLs and should be encoded:

  • & (separator) ‚Üí %26
  • = (assignment) ‚Üí %3D
  • ? (query start) ‚Üí %3F
  • # (fragment) ‚Üí %23
  • / (path separator) ‚Üí %2F
  • (space) ‚Üí %20 or +

üí° Common Mistakes

  • ‚ùå Encoding the entire URL (breaks structure)
  • ‚ùå Double-encoding (encoding already encoded text)
  • ‚ùå Forgetting to encode query parameter values
  • ‚úÖ Encode only the values, not the structure
  • ‚úÖ Use encodeURIComponent() for values
  • ‚úÖ Use encodeURI() for full URLs
