üîó What is URL Encoding?

URL Encoding (also known as percent-encoding) is a mechanism for encoding information in a Uniform Resource Identifier (URI). It converts characters that aren't allowed in URLs into a format that can be safely transmitted over the Internet.

The web uses the ASCII character set. Any character that isn't an unreserved ASCII character (A-Z, a-z, 0-9, and - _ . ~) must be encoded. This is done by replacing the character with a % followed by its two-digit hexadecimal value.

Why is it necessary?

  • Ambiguity: Characters like &, =, and ? have special meaning in URLs. If you want to use them literally (e.g., in a password), they must be encoded.
  • Safety: Spaces and unsafe ASCII characters (like <, >, ") can break URLs or confuse servers.
  • Data Integrity: Ensures that the data you send matches exactly what the server receives, regardless of browser or network interpretations.

üí° Common Use Cases

1. API Parameters

When sending data to REST APIs via query parameters, everything must be encoded. If you search for "C++ & Java", sending it as ?q=C++ & Java will break because & starts a new parameter. Encoded, it becomes ?q=C%2B%2B%20%26%20Java.

2. Redirect Return URLs

Authentication flows (OAuth) often need a return_url parameter. Since this is a URL inside a URL, the inner URL must be fully encoded to prevent the browser from misinterpreting where the first URL ends.

3. UTM Tags for Marketing

Marketers use UTM parameters to track clicks. If a campaign name contains spaces or emojis (e.g., "Summer Sale ‚òÄÔ∏è"), it must be encoded to ensure analytics tools capture it correctly.

4. Form Data

When browsers submit HTML forms using application/x-www-form-urlencoded, they automatically apply this encoding. This tool helps you debug raw form data or simulate form submissions.

‚ö° Tips & Tricks

Space Encoding: + vs %20

Spaces can be encoded as %20 (standard) or + (legacy, used in query strings). This tool defaults to %20 for maximum compatibility, but handles decoding both correctly.

encodeURI vs encodeURIComponent

In JavaScript development:

  • Use encodeURI() for full URLs. It preserves protocol/host separators like : and /.
  • Use encodeURIComponent() for values within a URL (like query parameters). It encodes everything including / and :.

Avoid Double Encoding

A common bug is encoding text that is already encoded, resulting in %2520 (encoded %) instead of %20. Use the "Decode" mode first if you're unsure.

‚ùì Frequently Asked Questions

Does this tool handle Emoji?

Yes. Emojis and unicode characters are first converted to UTF-8 bytes and then percent-encoded. For example, üòä becomes %F0%9F%98%8A.

Is my data private?

100% Client-Side. No data is sent to any server. All processing happens in your browser's memory, making it safe for internal URLs, tokens, or sensitive query strings.

Why does my decoded URL look wrong?

You might need to decode multiple times. Some systems apply encoding layers (e.g., a URL inside a JSON object inside a URL). Click "Decode" repeatedly until it looks readable.

Can I encode large blocks of text?

Yes, but keep in mind that URLs generally have a max length limit (often 2048 characters) in many browsers and servers. For larger data, consider using Base64 or POST requests instead.
